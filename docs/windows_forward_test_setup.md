# Windows PC（ノート）でMT5 EAフォワードテストを行う際の設定・注意点

## 目的

このドキュメントは、EA完成後にWindowsノートPCでMT5を起動し、デモまたは小ロット本番でフォワードテストを行う際の設定・注意点をまとめたもの。

フォワードテスト段階では、いきなりVPSへ移行せず、まずWindows PC上で以下を確認する。

- EAが意図通りにエントリーするか
- Entry success が出るか
- SL / TP が正しく設定されるか
- 方向が正しいか
- 手動決済が問題なくできるか
- ロット計算が正しいか
- 時間フィルタ・除外日フィルタが正しく機能するか
- MT5のログにエラーが出ていないか
- 通信断や再起動後に復帰できるか

---

## 基本方針

フォワードテスト初期は、以下の流れで進める。

```text
Windows PCでデモフォワード
↓
Windows PCで小ロット本番フォワード
↓
安定確認後にVPSへ移行
```

最初からVPSにすると、トラブル時に原因が切り分けにくい。

```text
EAの問題なのか
MT5の問題なのか
VPSの問題なのか
OANDA側の問題なのか
Windows側の問題なのか
```

を判断しづらくなるため、まずは手元のWindows PCで動作確認する。

---

## EA稼働に必要な条件

MT5のEAは、基本的に以下の条件が揃っている間だけ稼働する。

- Windows PCが起動している
- PCがスリープしていない
- PCが休止状態になっていない
- インターネットに接続されている
- MT5が起動している
- 対象口座にログインしている
- 対象チャートが開いている
- EAがチャートに適用されている
- MT5上部の「Algo Trading」がON
- EA側の自動売買許可がON

そのため、フォワードテスト中は特に以下に注意する。

```text
スリープしない
休止状態にしない
Windows Updateで勝手に再起動させない
MT5を閉じない
通信を切らさない
```

---

## Windowsの電源設定

### 1. スリープを無効化する

Windows 11の場合。

```text
スタート
→ 設定
→ システム
→ 電源とバッテリー
→ 画面とスリープ
```

以下のように設定する。

| 項目 | 推奨設定 |
|---|---|
| 電源接続時に画面をオフにする | 任意。30分、1時間、またはなし |
| 電源接続時にデバイスをスリープ状態にする | なし |

重要なのは、画面オフではなく、**スリープを「なし」**にすること。

画面が消えるだけならMT5は動く。  
PC本体がスリープするとMT5も止まる。

---

## ノートPCのフタを閉じてもスリープしない設定

ノートPCの場合、フタを閉じるとスリープになることがあるため、以下を確認する。

```text
コントロールパネル
→ ハードウェアとサウンド
→ 電源オプション
→ カバーを閉じたときの動作の選択
```

以下のように設定する。

| 項目 | 電源に接続 |
|---|---|
| カバーを閉じたときの動作 | 何もしない |
| 電源ボタンを押したときの動作 | スリープ以外を推奨 |
| スリープボタンを押したときの動作 | 何もしない、またはスリープ以外 |

ただし、ノートPCのフタを完全に閉じっぱなしにすると熱がこもる可能性がある。

フォワードテスト中は、できれば以下の状態が望ましい。

- 電源アダプター接続
- フタは少し開けておく
- 排熱しやすい場所に置く
- 布団やクッションの上に置かない
- 長時間高温にならないようにする

---

## 詳細な電源設定

以下も確認する。

```text
コントロールパネル
→ ハードウェアとサウンド
→ 電源オプション
→ 使用中プランの「プラン設定の変更」
→ 詳細な電源設定の変更
```

### スリープ

| 項目 | 推奨設定 |
|---|---|
| 次の時間が経過後スリープする | なし |
| ハイブリッドスリープを許可する | オフ |
| 次の時間が経過後休止状態にする | なし |
| スリープ解除タイマーの許可 | 任意 |

### ハードディスク

| 項目 | 推奨設定 |
|---|---|
| 次の時間が経過後ハードディスクの電源を切る | なし、または0分 |

### ワイヤレスアダプター

| 項目 | 推奨設定 |
|---|---|
| 省電力モード | 最大パフォーマンス |

Wi-Fi運用の場合、ワイヤレスアダプターの省電力設定で通信が不安定になる可能性があるため注意する。

---

## ネット接続

フォワードテストでは、できれば有線LANを推奨。

優先順位は以下。

```text
有線LAN
↓
安定した自宅Wi-Fi
↓
スマホテザリング
```

スマホテザリングでも動作確認はできるが、本番に近いフォワードテストでは不安定要素になる。

注意点。

- Wi-Fiが自動で切れないか
- ルーターが夜間に再起動しないか
- PCが省電力で通信を落とさないか
- テザリングの場合、スマホ側がスリープや省電力で通信を切らないか
- 回線断後にMT5が自動復帰するか

## Windows Update対策

### 基本方針

Windows Updateを完全停止するのではなく、**EA稼働中に勝手に再起動されないようにする**。

推奨運用は以下。

```text
平日：Windows Updateを一時停止
週末：手動でWindows Update → 再起動 → MT5/EA復帰確認
```

完全に永久停止するのはセキュリティ上おすすめしない。

---

### 更新の一時停止

Windows 11の場合。

```text
設定
→ Windows Update
→ 更新の一時停止
```

フォワードテスト期間中は、平日に勝手な更新・再起動が入らないように一時停止しておく。

---

### アクティブ時間の設定

```text
設定
→ Windows Update
→ 詳細オプション
→ アクティブ時間
```

EAを止められたくない時間帯をできるだけ長く設定する。

ただし、FXは平日ほぼ24時間動くため、アクティブ時間だけでは不十分。  
基本は、**更新一時停止＋週末手動更新**で管理する。

---

## 週末のWindows Update手順

週末に以下を行う。

```text
1. MT5の状態確認
2. 必要に応じてEAを停止
3. Windows Updateの一時停止を解除
4. 更新プログラムを確認
5. Windows Updateを実行
6. Windowsを再起動
7. MT5を起動
8. 口座接続を確認
9. Algo TradingがONか確認
10. EAがチャートに残っているか確認
11. エキスパートログ・操作履歴を確認
12. 月曜に向けてWindows Updateを再度一時停止
```

---

## MT5側の設定確認

### Algo TradingをONにする

MT5上部の **Algo Trading** ボタンをONにする。

OFFの場合、EAは稼働しない。

---

### エキスパートアドバイザ設定

MT5で以下を確認する。

```text
ツール
→ オプション
→ エキスパートアドバイザ
```

確認項目。

- アルゴリズム取引を許可
- 必要に応じてDLL使用を許可
- WebRequestが必要なEAの場合はURLを許可

EAによって必要な設定は異なるため、EA仕様に合わせて確認する。

---

## チャート側の確認

EAを稼働させるチャートで以下を確認する。

- 対象通貨ペアが正しい
- 対象時間足が正しい
- EAがチャートに適用されている
- パラメータが正しい
- 自動売買が許可されている
- チャート右上のEA状態が正常
- SL / TP設定が想定通り
- ロット設定が想定通り
- 時間フィルタが想定通り

---

## MT5再起動後の復帰確認

フォワード前に、一度わざとWindowsを再起動して確認する。

確認手順。

```text
1. Windowsを再起動
2. MT5が起動するか確認
3. 口座にログインできているか確認
4. 対象チャートが復元されているか確認
5. EAがチャートに残っているか確認
6. Algo TradingがONか確認
7. エキスパートログにエラーがないか確認
8. EAが正常に待機状態へ戻っているか確認
```

再起動後にEAが外れていたり、Algo TradingがOFFになっている場合は、本番運用前に原因を確認する。

---

## MT5をWindows起動時に自動起動する設定

万が一Windowsが再起動した場合に備えて、MT5をスタートアップに入れておく。

手順。

```text
1. Windowsキー + R
2. shell:startup と入力
3. Enter
4. 開いたスタートアップフォルダにMT5のショートカットを入れる
```

これでWindows起動時にMT5が自動起動する。

ただし、MT5が起動しても以下が自動復帰するとは限らない。

- 口座ログイン
- 対象チャート
- EA適用状態
- Algo Trading ON
- EAの内部状態

必ず事前に再起動テストを行う。

---

## フォワードテスト中の日次確認

最低1日1回、以下を確認する。

- MT5が起動しているか
- 口座接続が切れていないか
- Algo TradingがONか
- EAが対象チャートに残っているか
- エキスパートログにエラーがないか
- 操作履歴に注文エラーがないか
- 想定外のエントリーがないか
- 想定すべきエントリーを逃していないか
- SL / TPが正しく入っているか
- 手動決済が問題なくできるか

---

## フォワードテスト初期に確認する項目

EA完成後、まず以下を確認する。

### Entry success

- 指定時間にエントリーできたか
- エントリー条件成立時だけ発注したか
- エントリー条件不成立時に発注していないか
- ログにEntry successが出ているか

### SL / TP

- SLが想定位置に入っているか
- TPが想定位置に入っているか
- pips計算が正しいか
- JPYペアと非JPYペアで計算ズレがないか
- 5桁/3桁ブローカー仕様に対応できているか

### 方向確認

- Long条件でBuyしているか
- Short条件でSellしているか
- 通貨ペアごとの方向条件が逆になっていないか

### 手動決済

- 手動決済できるか
- 手動決済後にEAが再エントリーしないか
- 再エントリー禁止条件が必要か
- 手動決済後のログが正しいか

### 時間フィルタ

- 指定時間だけ稼働しているか
- 稼働対象外の時間に発注していないか
- サーバー時間と日本時間のズレがないか
- 夏時間の影響がないか

### 除外日フィルタ

- NFP
- CPI
- FOMC
- BOJ
- BOE
- RBA
- その他EA仕様上の停止日

上記の除外日に発注しないか確認する。

---

## ノートPC運用時の物理的な注意点

- 電源アダプターは常時接続
- バッテリー残量だけで運用しない
- フタを閉じっぱなしにしない
- 排熱できる場所に置く
- 布団、カーペット、クッションの上に置かない
- 夏場は高温に注意
- 家族が触らない場所に置く
- 誤って電源ボタンを押さないようにする
- LANケーブルや電源ケーブルが抜けないようにする

---

## 本番移行前の判断基準

以下を満たしてから本番運用、またはVPS移行を検討する。

- デモフォワードで意図通り動いた
- Entry successを確認した
- SL / TPが正しい
- 方向が正しい
- 手動決済が問題ない
- ロット計算が正しい
- 時間フィルタが正しい
- 除外日フィルタが正しい
- 再起動後の復帰を確認した
- Windows Update後の復帰を確認した
- 通信断後の挙動を確認した
- MT5ログに重大エラーがない
- 小ロット本番で問題が出ていない

---

## VPS移行の目安

以下に該当する場合は、VPS移行を検討する。

- 小ロット本番で安定した
- EAを常時稼働させたい
- 指定時間の取り逃しを避けたい
- Windows PCを常時起動するのが不安
- 自宅回線が不安定
- 停電・再起動・スリープ事故を避けたい
- 複数通貨・複数EAを運用する
- 本番資金を入れて継続運用する

候補としては、FX/MT5用途なら以下。

```text
シンクラウドデスクトップ for FX
ConoHa for Windows Server
MT5公式VPS
```

完成EAの安定運用では、最終的にVPSを使う方が安全。

---

## まとめ

Windows PCでのフォワードテストは、EA完成直後の動作確認として有効。

ただし、以下は必ず設定・確認する。

```text
スリープなし
休止状態なし
フタを閉じてもスリープしない
Windows Updateの自動再起動対策
MT5自動起動設定
Algo Trading ON
再起動後の復帰確認
日次ログ確認
週末の手動Windows Update
```

フォワードテスト初期はWindows PCで原因を切り分ける。  
安定確認後、本番継続運用ではVPSへ移行する。
