# Time Entry Portfolio Lab 開発ログ

## 目的

本レポジトリは、時間固定エントリー・時間固定決済を中心としたFXポートフォリオ戦略を管理・検証するための開発ログである。

現在の主な目的は以下。

- 現行運用中の時間手法ポートフォリオをPythonで再現する
- 各ロジックの条件、除外日、SL/TP、スプレッド設定を明文化する
- バックテスト結果と実運用結果の差異を確認する
- 追加フィルタの検証により、PF・DD・RoMDの改善可能性を調査する
- 過剰最適化を避けるため、IS/OOS・年別・月別・ロジック別に安定性を確認する

---

## 現在の対象ポートフォリオ

### マスター16ロジック

対象通貨ペア：

- USDJPY
- EURJPY
- GBPJPY
- AUDJPY

主な特徴：

- JST基準の固定時間エントリー
- JST基準の固定時間決済
- SL/TPあり、またはTPなし時間決済
- 年末年始停止
- 重要指標日停止
- 月別・日付別の個別停止条件あり

### 追加オージー絡みロジック

対象通貨ペア：

- AUDUSD
- AUDJPY
- EURAUD
- GBPAUD

主な特徴：

- 中国実需狙い
- AUDUSDは毎月9日〜15日、および25日〜月末も稼働
- RBA、豪CPI、FOMC、ECB、BOEなどのイベント停止を使用
- FOMCは前日・当日停止が必要

### EA / GA追加ロジック

対象通貨ペア：

- EURAUD
- GBPAUD

主な特徴：

- JST固定時間エントリー
- 複数曜日・複数時間帯
- 翌日持ち越しロジックあり
- 指標停止と季節性停止を使用

## 検証エンジンに関する注意点

### 1. スプレッド処理

現行コードでは、エントリー価格を不利にすることでスプレッドを反映している。

```python
ep = o[s_idx] + spread if is_long else o[s_idx] - spread
```

この方式は、トレードごとにスプレッド分を1回控除する考え方に近い。

今後の改善候補：

- `entry_adjust`方式として明記する
- Bid/Ask方式に変更したバージョンと比較する
- 通貨ペアごとのスプレッド設定を外部CSVで管理する

---

### 2. SL/TP同一足内判定

現行コードでは、1分足の同一足内でSLとTPの両方に到達した場合、SLを優先する。

これは保守的な判定であり、検証上は安全寄り。

今後の方針：

```text
same_bar_policy = sl_first
```

として明記する。

---

### 3. 指標フィルター

現行コードでは、基本的にエントリー日の日付で指標停止を判定している。

ただし、仕様上は以下が望ましい。

```text
Entry日時〜Exit日時の保有期間に重要指標日が含まれる場合は見送り
```

特に翌日持ち越しロジックでは、エントリー翌日の指標を見逃す可能性がある。

今後の改善候補：

- エントリー日だけでなく、保有期間内の日付をチェックする
- 可能なら指標発表時刻も管理する
- 最低限、保有期間内にイベント日が含まれるかを判定する

---

### 4. FOMC前日停止

オージー絡みロジックでは、FOMC当日だけでなく前日も停止する仕様。

今後の実装方針：

```text
FOMC_STOP_DATES = FOMC当日 + FOMC前日
```

としてイベントフィルターに追加する。

---

### 5. 2026年カレンダー

2026年の経済指標カレンダーは別途保有あり。

今後の方針：

- 2026年イベント日を追加
- 可能ならイベント日をPythonコード直書きではなくCSV管理へ移行する
- 年次更新しやすい形にする

### v1.1での主な修正内容

#### 1. UJ_Short_Coreを本命仕様へ修正

既存コードでは、UJ_Short_Coreの通常日エントリーが `07:53` になっていた。

正しい仕様は以下。

```text
通常日Entry：08:04
通常日SL/TP：SL50 / TPなし
```

また、ゴトー日は以下。

```text
ゴトー日Entry：09:55
ゴトー日SL/TP：SL20 / TP50
```

v1.1では、日付によりEntry時刻・SL/TPを分岐するように修正した。

---

#### 2. UJ_Short_Coreの末日停止はカレンダー末日を正式採用

比較用コードでは以下2パターンが存在していた。

```text
12A：カレンダー末日停止
12B：最終営業日停止
```

本命版では、ユーザー判断により以下を正式採用した。

```text
カレンダー末日停止
```

そのため、12A/12Bの二重計上は廃止し、本命ロジックは以下のみとする。

```text
12_UJ_Short_Core
```

---

#### 3. UJ_Fix_MidWeekの曜日を水曜・木曜で維持

Pythonの `weekday()` は以下。

```text
月曜=0
火曜=1
水曜=2
木曜=3
金曜=4
```

仕様では「毎月25日以降の水曜・木曜」であるため、v1.1では以下を採用する。

```python
wd in [2, 3]
```

---

#### 4. SL/TP同一足判定はSL優先を維持

1分足の同一足内でSLとTPの両方に到達した場合、保守的にSLを優先する。

```text
same_bar_policy = sl_first
```

この仕様は、PFを過度に良く見せないための安全側判定として維持する。

---

#### 5. スプレッド処理は現行方式を維持

現行コードでは、エントリー価格を不利にずらすことでスプレッドを反映している。

```text
spread_mode = entry_adjust
```

Longの場合：

```python
entry_price = open + spread
```

Shortの場合：

```python
entry_price = open - spread
```

v1.1では既存結果との比較を優先し、この方式を維持する。
